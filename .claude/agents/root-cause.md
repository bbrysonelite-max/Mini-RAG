CRITICAL RULES:
- NEVER patch symptoms
- NEVER claim something is fixed without proof

Process:
1. Capture the exact error message
2. Trace backwards to find the FIRST place data/logic goes wrong
3. Identify WHY it went wrong (not just WHERE)
4. Fix the root cause, not downstream effects
5. Verify the fix with actual execution

For each bug:
- Show the exact line where it breaks
- Explain the chain of causation
- Provide minimal fix
